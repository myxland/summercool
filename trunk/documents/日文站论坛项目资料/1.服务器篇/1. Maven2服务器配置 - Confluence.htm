1. Maven2的安装

    A. 首先获取Maven2工具包，可以从SVN上获取，地址为http://svn.japan.alibaba-inc.com/repos/ali_jp/club/maven2

    B. 选择存放Maven2的本地地址，如：D:\maven2。

        

        

    C. 下面将Maven2配置到系统的环境变量中。

    D. 打开系统的环境变量设置界面，点击新增按钮，输入变量名为M2_HOME，变量值为Maven2的存放路径如：D:\maven2，点击确定。

       

    E. 点击新增按钮，输入变量名为M2，变量值为%M2_HOME%\bin，点击确定。

       

    F. 添加到PATH中，选择PATH，点击编辑，在变量值后添加%M2%，注意和前面的值用;分隔。

       

    G. 确保环境变量有JAVA_HOME变量，值为JDK所在位置，如：D:\jdk1.5.0_16，如果没有请添加。

    H. 打开命令提示符工具，输入mvn -version，回车，如果返回Maven，Java和系统版本信息，说明Maven2安装成功，否则请检查上面的过程。

       

    I. 现在Maven2已经安装成功了。

2.Maven2资源库的配置

    A. 下面开始配置Maven2的资源库。

    B. 找到Maven2的全局配置文件settings.xml，在Maven2工具包下面的conf文件夹中，如：D:\maven2\conf\settings.xml。

       

    C. 打开settings.xml文件。

    D. 找到localRepository节点，将值修改为你的计算机中Maven2的资源库地址；本地Maven2的资源库地址在Maven2工具包下面的conf文件夹的repository中。

         

<!-- localRepository
 | The path to the local repository maven will use to store artifacts.
 |
 | Default: ~/.m2/repository
<localRepository>/path/to/local/repo</localRepository>
-->
<localRepository>D:\maven2\repository</localRepository>

    E. 如果当本地的资源库不存在需要的资源包，而又不希望去Maven2的网络资源库下载，可以设置offline的值为true。

<!-- offline
 | Determines whether maven should attempt to connect to the network when executing a build.
 | This will have an effect on artifact downloads, artifact deployment, and others.
 |
 | Default: false
<offline>false</offline>
-->
<offline>true</offline>

    F. 保存settings.xml，现在Maven2资源库已经配置成功了。

3.Maven2其他配置

    A. 编码配置(论坛项目主配置文件：$path$\japan-club-package\pom.xml)。

<build>
    <plugins>
        <plugin>
            <artifactId>maven-compiler-plugin</artifactId>
            <configuration>
                <source>1.5</source>
                <target>1.5</target>
                <encoding>utf-8</encoding>
            </configuration>
        </plugin>
    </plugins>
</build>

      说明：1. 找到项目的主pom.xml配置文件，一般在项目的根目录下。
                2. 添加上面代码，以覆盖父pom配置信息。修改maven-compiler-plugin插件的JDK编码和版本。

    B. 配置web项目的源目录和输出目录(论坛项目主配置文件：$path$\japan-club-package\pom.xml)。

<build>
    <plugins>
        <plugin>
            <artifactId>maven-war-plugin</artifactId>
            <configuration>
                <warSourceDirectory>src/main/webapp</warSourceDirectory>
                <outputDirectory>target</outputDirectory>
            </configuration>
        </plugin>
    </plugins>
</build>

      说明：1. 找到项目的主pom.xml配置文件，一般在项目的根目录下。
                2. 添加上面代码，以覆盖父pom配置信息。修改maven-war-plugin插件的源目录和输出目录。可以根据需要修改warSourceDirectory和outputDirectory的值。

    C. 配置web项目的源目录和输出目录应用举例，如下：

<plugin>
    <artifactId>maven-war-plugin</artifactId>
    <configuration>
        <warSourceDirectory>${basedir}/webapp</warSourceDirectory>
        <outputDirectory>${packagedir}</outputDirectory>
    </configuration>
</plugin>

      说明：1. 上面配置信息是论坛项目中主配置文件：$path$\japan-club-package\pom.xml的配置片段。
                2. 变量$basedir$是maven2中自带的内部变量，含义是：当前项目的根目录的意思；
                    注：因为大部分网上的资料都说Maven2不支持标准的web结构，其实是不对的；因为通过上面的配置就可以使maven2的web工程支持标准的web结构。
                3. 变量$packagedir$是项目打包时生成war包的路径。

    D. 配置Java源代码和测试代码的位置。

<build>
    <!-- Java 源代码位置 -->
    <sourceDirectory>src/main/java</sourceDirectory>

    <!-- Java 测试代码位置 -->
    <testSourceDirectory>src/test/java</testSourceDirectory>

    <!-- Java 输出位置 -->
    <outputDirectory>target/classes</outputDirectory>

    <!-- Java 测试输出位置 -->
    <testOutputDirectory>target/test-classes</testOutputDirectory>
</build>

      说明：1. 找到项目的主pom.xml配置文件，一般在项目的根目录下。
                2. 添加上面代码，以覆盖父pom配置信息。修改Java源代码和测试代码的源目录和输出目录。可以根据需要修改各值。
